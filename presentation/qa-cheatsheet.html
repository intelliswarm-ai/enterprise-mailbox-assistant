Anticipated Questions

CREDIBILITY
How did 5 people build this in 6 months?
Key message: "We leveraged, we didn't reinvent."
  • LangGraph + CrewAI gave us agent orchestration out of the box
  • Docker Compose eliminated infrastructure work
  • Open-source ML (scikit-learn) reduced training time
  • Angular 18 + NgRx provided enterprise patterns
  • Public APIs (SEC EDGAR, OFAC) - no data sourcing needed
CREDIBILITY
Why only 82% accuracy? Banks need higher.
Answer: "82% is the ML layer. The multi-agent validation layer adds human-like judgment on top, achieving 95%+ on investigated cases. We optimized for precision over recall - in banking, false positives are worse than missing edge cases. The system flags uncertain cases for human review."
TECHNICAL
How does the offline/on-premise mode work?
Answer: "We use Ollama with Phi3/TinyLlama as automatic fallback. When OpenAI is unavailable or disabled, the system seamlessly switches to local inference. Zero code changes required. This is critical for Swiss data sovereignty - customer data never leaves your infrastructure."
TECHNICAL
What happens if an external API fails?
Answer: "Every tool has automatic fallback. OFAC uses cached data, search falls back to mock data, web scraping falls back to HTTP. The system never hard-fails. We designed for resilience from day one."
TECHNICAL
What LLMs do you use?
Answer:
  • Primary: OpenAI GPT-4o-mini (fast, cost-effective)
  • Fallback: Ollama with Phi3 or TinyLlama (fully local)
  • ML Models: Naive Bayes (76.5%), Random Forest (71.7%), Fine-tuned transformer (59%)
  • Ensemble: High-precision OR voting = 82.1%
BUSINESS
What's the cost per email?
Answer:
  • With OpenAI: ~$0.001 per email (GPT-4o-mini pricing)
  • With Ollama: Free (local compute only)
  • 10,000 emails/day: ~$10/day with OpenAI, $0 with Ollama

Is this FINMA compliant?
Answer: "The foundation is there - full audit trail, OFAC screening, AML/KYC risk assessment, on-premise deployment. Full FINMA compliance requires bank-specific customization (data retention policies, specific reporting formats). We built the compliance infrastructure - the bank adds their specific rules."

  • Email server integration (2-3 days)
  • Custom compliance rules (1-2 weeks)
  • Security hardening (1 week)
  • User training (ongoing)
"
BUSINESS
What makes this different from existing solutions?
Answer:
  • Multi-agent collaboration - not just detection, but investigation
  • Swiss banking focus - OFAC, AML/KYC built-in
  • On-premise first - data sovereignty by design
  • Explainable decisions - full audit trail of agent reasoning
  • Extensible platform - add tools/teams without code changes
TECHNICAL
What about false positives? How do you handle them?
Answer: "We optimized for precision (81.9%) to minimize false positives. Uncertain cases get human-in-the-loop review. The multi-agent layer adds a second check - if agents disagree, it's flagged for manual review. Over time, feedback improves the models."
TECHNICAL
Can you add new types of analysis/agents?
Answer: "Yes, by design:
  • New tool: 30 minutes (implement interface, register)
  • New agent: 1 hour (define role, persona, tools)
  • New team: 2 hours (4 agents + collaboration pattern)
The pluggable architecture was a core design goal."
CREDIBILITY
Why should we trust AI for critical banking decisions?
Answer: "We don't replace human judgment - we augment it. The system:
  • Triages 10,000 emails to the 100 that need attention
  • Provides context - agent analysis as decision support
  • Flags uncertainty - humans review edge cases
  • Creates audit trail - every decision is explainable
It's AI + Human, not AI vs Human."
